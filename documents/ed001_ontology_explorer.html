6 SWRL rules implement the automated reasoning pipeline. Rules are serialized as swrl:Imp individuals. A reasoner (Pellet, HermiT) fires them over the ABox to infer new facts.

Rule 1 — Stress detection
IF: hasEmotion(learner, AnxietyState) ∧ isAttempting(learner, task) ∧ hasContext(task, ctx) ∧ taskDifficulty(ctx, "High") ∧ timePressure(ctx, true)
THEN: hasCognitiveState(learner, HighCognitiveLoad)
WHY: Combines an emotional signal (anxiety) with objective context features (hard task + time pressure) to infer cognitive overload. This is the entry point of the stress cascade — Rule 2 then fires on this output.
Rule 2 — High cognitive load → impulsive decision + high risk
IF: hasCognitiveState(learner, HighCognitiveLoad)
THEN: hasDecisionStrategy(learner, ImpulsiveDecision) ∧ hasRisk(learner, HighRisk)
WHY: Applies Dual Process Theory. When working memory is saturated, System 2 (deliberative) is unavailable, forcing fallback to System 1 (heuristic/impulsive). Also flags HighRisk to activate Rule 4's intervention pipeline.
Rule 3 — Curiosity + focused attention → exploratory reasoning + low risk
IF: hasEmotion(learner, CuriosityState) ∧ hasCognitiveState(learner, FocusedAttention)
THEN: hasDecisionStrategy(learner, ExploratoryReasoning) ∧ hasRisk(learner, LowRisk)
WHY: Models the "flow state" — the optimal learning condition. Curiosity with focused attention predicts deliberate exploration of the solution space and minimal error probability.
Rule 4 — Anxiety + high risk → stress reduction recommendation
IF: hasEmotion(learner, AnxietyState) ∧ hasRisk(learner, HighRisk)
THEN: receivesRecommendation(learner, StressReduction)
WHY: Closes the intervention loop after Rules 1→2 have identified a stressed, high-risk learner. StressReduction maps to practical interventions: break task into chunks, remove time pressure, add reassuring feedback.
Rule 5 — Boredom → engagement boost recommendation
IF: hasEmotion(learner, BoredomState)
THEN: receivesRecommendation(learner, EngagementBoost)
WHY: Boredom alone is sufficient to trigger intervention — no additional conditions needed. EngagementBoost maps to: increase task difficulty, add gamification elements, introduce novelty.
Rule 6 — Frustration + distraction → avoidance strategy + high risk
IF: hasEmotion(learner, FrustrationState) ∧ hasCognitiveState(learner, DistractedState)
THEN: hasDecisionStrategy(learner, AvoidanceStrategy) ∧ hasRisk(learner, HighRisk)
WHY: Detects the dropout precursor pattern. Frustration (blocked goal progress) combined with divided attention predicts that the learner will start skipping tasks rather than engaging with them. HighRisk triggers ScaffoldingSupport via downstream intervention logic.
